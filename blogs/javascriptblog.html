How JavaScript Works?

december 2020 2 min read

So as a newbie who gets mesmerized with the beauty of javascript generally starts coding with javascript without the knowledge of the basics. At first, it seems really good, but this mesmerization with the language can become a myth, and you may start hating it. So let’s start with the basics of how the javascript works. Understanding how this works makes reading and writing code easier and less frustrating and allows you to focus on your logic rather than the grammar of the language.

HOW DOES IT WORK?

JavaScript is a single-threaded synchronous language. It means that the code is executed in a line by line fashion and that too with a specific direction.

So understand this everything happens inside the execution context. You can think of this as a container in which the whole javascript code is executed.

So this execution context consists of two components

Memory Component.

Code Component.

Memory Component: — It is also known as a variable environment. It consists of all the variables and functions in a key-value pair manner.

Code Component:- Also known as Thread of execution. In which the whole code is executed is line by line.

Call Stack

A call stack is a mechanism to keep track of its place in a script that calls multiple functions(execution context).

It tells which programs are currently executed and also tells which function is called within that function.

  • When a program calls a function that function is been pushed into the stack.
  • When a function is called within a function then this function is also put on the stack and is run when they are being called.
  • Once a function is over they are popped from the stack.
  • If the space allotted to them is more than the stack can accommodate, it results in stack overflow error.
